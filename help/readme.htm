AlegrDiff Version 1.0

AlegrDiff compares text and binary files. It allows also to merge the text differences.

The program allows to compare single files as well as the whole directory with subdirectories. The differences are shown with different font and color.

The files are compared either as text or binary.

To compare just two files, select File/Compare Files command in the menu, or click toolbar button. You will be prompted for the first file, then for the second file. The file open dialogs have Recent Dirs combobox, which allows you to jump to one of recent folders you used, and Binary comparision checkbox:

 

To compare directories, select File/Compare Directories... command in the menu, or click toolbar button. In the following dialog you can type in the directory names, or, by clicking on the [...] buttons, select them from the directory tree, or select from recently used folders.

You can specify pattern for the filenames to be included in comparison, in the Filename filter edit box. You can specify one or a number of patterns, separated by comma or by semicolon.

By clicking Advanced >> button, you can also specify some options, normally available in Preferences dialog. These are filename patterns which are recognized as C/C++ files, filename patterns which are  recognized as binary files, and filename patterns which will be excluded from compare, and tab indent.

As you click OK, the program compares all the files in the first directory against the corresponding files in the second directory, and then shows the comparison report, with File Name, Subdirectory, 1st Modified at, 2nd Modified at, Comparison result columns. You can sort the report or change sorting direction by clicking on the column header.

File menu and right-click context menu allow also to open the selected files in the default application registered for the file extension (equivalent of double-clicking the file in the Windows Explorer), and copy the selected files to any folder:

To see the detailed differences between two files, double click on the difference report line. The program allows to keep several files open. The file window can be closed by pressing Esc key.

The files will be shown just like in the text editor. The text which is only in the first or second file is shown with different style and color. Default for erased text is strikethrough, red, for added text is underlined, blue. You can change this in the Preferences dialog. You can move to the next difference by clicking button, or by pressing F7 key. You can move to the previous difference by clicking button, or by pressing Shift+F7 key.

You can use most of standard keyboard and mouse navigation commands. Mouse wheel is also supported. You can find a selected text or word under caret by pressing Ctrl+F3, then find the next or previous occurrence by pressing F3 or Shift+F3. You can select the text and copy it to the system clipboard.

Toolbar button shows line numbers along with the text. Numbers of lines which are present in both files are shown in black, numbers of first-file-only lines are shown in red (or whatever color is chosen for the erased text), numbers of second-file-only lines are shown in blue:

Toolbar button hides whitespace difference. Not all spaces are ignored, but only those which don't change the text syntax and meaning. See, for example, the original text:

 

And the text with the extra spaces hidden:

 

Extra spaces that are erased from the first file are not shown; extra spaces which are added to the second file are shown, but not marked as added. Spaces that break some reserved character sequences (+=, -=, and other C/C++ operators, and also spaces inside character and string constants are not extraneous, and are shown in the both cases. Of course, the reserved sequences and constants are recognized only in the files which bear some specific file name extensions.

The differences can be marked as accepted or declined. These attributes are used for merging the files. This feature allows to select only those changes that you like, and create a new file containing only selected changes. When the difference is marked, it is shown with different background color. The text that won't go to the merged file is shown with gray background; the text which goes to the merged file is shown with green background:

These colors can be customized in Preferences dialog.

To accept the difference, set the caret on it and select Merge/Accept menu command, or click toolbar button, or press Shift+F6. To decline the difference, set the caret on it and select Merge/Decline menu command, or click toolbar button, or press F6. You can also do the operation over all the differences in the selected text area. Text selection is done by the standard keyboard and mouse means. Accept/Decline commands are also available in the right-click context menu.

You don't need to mark all changes. The program will ask you what to do with unmarked ones - either accept or decline them. For example, if you want to base the merged file on the second one, you just need to decline some changes, and accept all others, when asked.

To save the merged file, select File/Merge Files And Save... menu command, or click toolbar button. You may be asked whether you wish to accept or decline all unmarked changes. You will then be asked for the file name. The saved resulting file will be open in the separate window in the program. You can then open it in the default editor, by selecting File/Open first file command.
